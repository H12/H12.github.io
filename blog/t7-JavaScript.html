JavaScript vs. Ruby

May 6th, 2015

JavaScript and Ruby are two tremendously popular languages. Each have their own unique strengths, weaknesses, and quirks which people either love or hate. Anyone with an amount of familiarity with these two languages know that the differences between them are abundant. In this blog post, I'm going to attempt to articulate some of the more significant differences.

I think a good place to start would be to describe some of the most common uses for JavaScript and Ruby. For the former, this is easy: JavaScript is built to be run by web browsers, and is used to bestow interactivity to web pages. If you ever come across a web page with a bunch of fancy animations or that allow you to interact with the web page in fun/interesting ways, odds are that website is using JavaScript. More simply, any time you're on a web-page and a pop up window appears asking you to input information or press a button, that pop-up is the result of the browser running a JavaScript method. Ruby, on the other hand, has a much less specific and more expansive array of uses. Ruby is a language that deals with data as a whole, as opposed to data that exists within a web-page. As a result, it has a much more exhaustive array of methods and plug-ins that help coders deal with different types of data. It has a great deal of flexibility, both in how it can be used, as well as in how it can be written.

To that end, the syntax for these two languages are considerably different as well. JavaScript has a much more rigid structure, using brackets, semicolons, parentheses, and other symbols to break things up. Ruby, in a manner of speaking, is much "cleaner". Where JavaScript would want you to put parentheses, Ruby is happy without them. Ruby's "cleanliness" plays a large role in its popularity. Getting stuff done requires markedly fewer keystrokes than many other languages, which makes it a very appealing option to lazy coders like myself.
